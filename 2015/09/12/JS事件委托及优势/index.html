JS事件委托及优势

2015-09-12
Javascript>Js事件委托
评论

Js事件委托也称为事件代理，就是事件冒泡原理，将本来发生在自己身上的事件让父级元素处理。例如，ul标签下有很多li，只需要让一个ul元素进行处理。

优势一：提高性能

#html
<ul id='ul'>
    <li>这是第1个li</li>
    <li>这是第2个li</li>
    <li>这是第3个li</li>
    <li>这是第4个li</li>
    <li>这是第5个li</li>
</ul>

#js 不使用事件委托
ul=ul.document.getElementById('ul');
li=ul.document.getElementsByTagName('ul');
for(var i=0;i<li.length;i++){
    li[i].onmouserover=function(e){
        var e=e||window.event;
        e.style.background='red';
    }
    li[i].onmouseout=function(e){
        var e=e||window.event;
        e.style.background='';
    }
}

#js 使用事件委托
ul=document.getElementById('ul');
ul.onmouseover=function(e){
    var a=e||window.event
    var target=e.target||e.srcElement;
    if(target.nodeName.toLowerCase()=='li'){
        target.style.background='red';
    }
}
ul.onmouseout=function(e){
    var a=e||window.event;
    var target=e.target||e.srcElement;
    if(target.nodeName.toLowerCase()=='li'){
        target.style.background='';
    }
}

两段代码对比可见，使用事件委托少了for循环，如果网页中li特别多的话，for循环所占用的空间就越多。

优势二：后添加的元素也有相同的事件

加入往ul里面再手动添加li，这些新添加的li与原有的li有相同的事件。

#html
<ul id="ul">
    <li>这是第1个li</li>
    <li>这是第2个li</li>
    <li>这是第3个li</li>
    <li>这是第4个li</li>
</ul>
<input type="button" value="添加" id="btn" />

#js

ul=ul.document.getElementById('ul');
li=ul.document.getElementsByTagName('ul');
#使用for循环添加事件
for(var i=0;i<li.length;i++){
    li[i].onmouserover=function(e){
        var e=e||window.event;
        e.style.background='red';
    }
    li[i].onmouseout=function(e){
        var e=e||window.event;
        e.style.background='';
    }
}

btn=document.getElementById('btn');
btn.onclick=function(){
    var ul=document.getElementById('ul');
    var newLi=document.createElement('li');
    newLi.innerHTML="I'm new li element";
    ul.appendChild(newLi);
}

为什么for循环就不能让新添加的元素有原有的元素一样的事件呢？因为程序是从上往下依次执行的，后面新添加的li元素不在for循环里面。而使用事件委托借助冒泡原理，只要新添加的元素在父级元素里面，父级元素就能收到子元素的事件，从而进行事件处理。

总结

  1. 这里使用的DOM0级事件处理程序
  2. 事件源(e.target||e.srcElement)：就是触发事件的本人。
  3. 事件委托，对于重复的新添加节点的应用可以大大提高性能。
