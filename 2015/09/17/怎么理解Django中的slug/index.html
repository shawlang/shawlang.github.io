怎么slugify文章的标题？

2015-09-17
Django 101
评论

在使用Django中DetailView这个类的时候，有属性是slug_field,slug_url_kwarg。Django中还内置了一个函数slugify，这个函数的作用是这样的：

Converts to ASCII. Converts spaces to hyphens. Removes characters that aren’t alphanumerics, underscores, or hyphens. Converts to lowercase. Also strips leading and trailing whitespace.
slugify(value),If value is “Joel is a slug”, the output will be “joel-is-a-slug”.

slugify标题对SEO非常有帮助。把有意义的标题放到url中，搜索引擎就知道抓取链接就更加精准。但是url中不能有空格，所以slugify总的来说就是把单词之间加上连字符（-）。但是我不理解的是用来传值的属性slug_url_kwarg偏要带上slug这个单词，而不是keywords之类的。

　slugify文章标题

假如标题是title，slugify(title)后就是一个新的值。如果需要标题slugify后的值，有两种方法，要么每次请求title的时候先调用slugify方法，即时转换；要么把slugify后的值存到数据库中，这样需要slugify后的值直接到数据库去取就可以了。加入文章很多，网站流量很大，每一次都要调用函数效率不高；所以一般采取第二种，为此Django还特地在Model里面增设了一个SlugField字段。

#models.py
from django.template.defaultfilters import slugify

class Post(models.Model):
    title=models.CharField(max_length=30)
    slug=models.SlugField(null=True,blank=True)

    def save(self,*args,**kwargs):
        if not self.id:  #这里有人建议设置这个，让每篇文章的标题只slugify一次。
            self.slug=slugify(self.title)
        return super(Post,self).save(*args,**kwargs)

因为django中字段中有AutoField id，也就是有auto_increment字段，所以models.py中的类都会有默认的save()函数。如果想在Django中自动的slugify每一个post的标题，只需要重载save()函数。

注意如果DetailView有个slug_field属性，且默认值是slug，所以如果在model里面设置的model.SlugField()的字段名不是slug,那么DetailView中的slug_field要做相应的改变。

扩展：slugify后台admin某些字段

class PostAdmin(admin.ModelAdmin):
    prepopulated_fields={'slug':('name',)}

admin.site.register(Post,PostAdmin)
