怎么基于Class Based View中的ListView添加分页功能

2015-09-16
Django 101
评论

Django中使用函数式编程来添加分页，已经很简单了，但是基于Class Based View更加简单。

#views.py
from django.views.generic import ListView

class IndexListView(ListView):
    model=Post
    context_object_name='posts'
    template_name='index.html'
    paginate_by=2  #每页2篇文章

#index.html

{ % if is_paginated % }
   <div class="pagination">
   { % if page_obj.has_previous % }
        <span>
            <a href="?pages={ {page_obj.previous_page_number} }">上一页</a>
        </span>
   { % endif % }
        <span>
            第{ {page_obj.number} }页/总共{ {page_obj.paginator.num_pages} }页
        </span>
    { % if page_obj.has_next %}
        <span>
            <a href="?page={ {page_obj.next_page_number} }">下一页</a>
        </span>
    { % endif % }
    </div>
{ % else % }
    <p>暂时还没有文章</p>
{ % endif % }

Class Based View中ListView关于分页的常用属性和方法

  1. paginate_by ：int 每页的个数

  2. paginate_orphans : int 一般分页之后，最后一页可能小于每一页的个数，这时候利用这个参数将最后一页的剩余几个补到倒数第2页。

  3. page_kwarg: string（default page_kwarg=page）。就是通过request.get获取当前是第几页的变量，如果shawlang.com/posts?[page]。默认是page，想修改的话可以通过这个参数来修改，注意如果这里修改了模板中a标签href也要修改。

  4. paginator_class：用函数式编程，通常要导入Paginator这个类。CBV形式的编程通过这个来设置，默认的就是平常导入的类：django.core.paginator.Paginator
