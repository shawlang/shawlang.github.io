Django做博客如何显示文章摘要

2015-09-08
Django>Django过滤器
评论

博客文章常要显示到分类或者首页，很多时候不需要全部显示出来，只是需要显示一个摘要。当然为了更好的SEO，单独在Models.py里面建立一个字段，每次都手动添加摘要是再好不过了。不过想偷点懒，咋办呢？

Django模板中有很多的有用的过滤器，比如文章字符截取过滤器：truncatewords,truncatechars,truncatewords_html,truncatechars_html。

一、Truncatewords 和 Truncatewords_html

1.例如article.body是这样的： 国外框架Django is a powerful framework 

article.body|truncatewords:3
# output: 国外框架Django is a ...

可见：truncatewords对中文不奏效

2.article.body:

国外框架Django is a powerful framework 

article.body|truncatewords_html:3
# output: <p>国外框架Django is a ...</p>

可见：truncatewords_html 会自动把文章内容中没有闭合的标签自动闭合。（看起来有点不友好）

3.article.body:

国外框架Django is a powerful framework

这是我的第一篇文章 

article.body|truncate_html:100|safe
#output： 
国外框架Django is a powerful framework
这是我的第一篇文章

可见：truncatewords_html后面再用safe过滤器，会自动转换文章中的html标签。

二、Truncatechars 和 Truncatechars_html

例如：article.body是这样的： 国外框架Django is a powerful framework 

article.body|truncatechars:8
output: 国外框架D...  #这里'...'也是算在整个截取的字符数内的。

可见：truncatechars支持中文截取，但是这又英文支持不太好，有些单词被截取过后变成无意义的字符。

其他Truncatechars_html和safe过滤器都与truncatewords是一样地使用。

三、自定义过滤器

由上面实验的结果看，truncatechars对英文支持不太好，truncatewords对中文支持不太好。要是过滤器碰到英文的时候按照单词来截取，碰到中文的时候按照字符来截取就好了。
