使用get_absolute_url函数改变模板超链接硬编码

2015-09-18
Django101
评论

在Django模板中遇到超链接的时候，初学的时候通常这样写：href=’/posts/{ {post.id } }’。如何需要将urls.py中posts变为articles，就需要将模板中很多的超链接中的posts修改为articles，超级麻烦！所以，能不能将posts变为像post.id一样的变量呢？

Django中有一个get_absolute_url的方法，可以返回数据对象的绝对地址。

#urls.py
urlspatterns=patterns[]
      urls(r'^posts/(?P<post_id>)$',PostDetailView,name='post_detail'),
    ]

#在models.py中数据模型中定义
#models.py
class Post(models.Model):
    title=models.CharField(max_length=50,verbose_name='标题',default='')

    def get_absolute_url(self):
        from django.core.urlresolvers import reverse

        return reverse('post_detail',args=str(self.id))

#在模板template中的使用

<p><a href="{ {post.get_absolute.url} }/{ {post.id} }">文章id</a></p>

为什么get_absolute_url函数在模板中post对象能够使用？

模板中post对象，说白了就是通过views.py中的函数或者类，将数据模型中Post类实例化的对象。所以models.py中定义一个sayHello()函数，模板中post对象也能够调用，只是django模板中对象的方法和属性一样，使用的时候不需要括号。

函数reverse的简单解释

在get_absolute_url函数引入了一个reverse函数，第一参数是urls.py中定义的name值。（Django文档中称为viewname，这里有点不理解），第二个参数是就是urls.py中路径定义的需要传递的参数值。注意这里参数都要变成字符串类型。

通过reverse的作用可以看出这个函数为什么称为“reverse”了，在urls.py进行相应的配置使得路径找到相应的view函数或者类，而通过reverse函数找到对应的路径。

#例子
python manage.py shell 进入shell窗口

from django.core.urlresolvers import reslove,reverse
reverse('post_detail',args='2')
#output: /posts/2

resolve函数的简单解释

通过resolve函数可以找到reverse的output:’/posts/2’的name,args,kwargs,func。也就是整个urls.py三个配置参数。

#例子
match=resolve('/post/2')
match.url_name
# output: post_detail
match.func
# output: blog.views.PostDetailView
match.args:
#output: {'post_id': u'2'}

#一次性得到三个参数
func,args,kwargs=resolve('/posts/2')

总结

  1. 初步了解 reverse,resolve的作用，还没有深入应用

  2. 定义数据模型的时候，定义好get_absolute_url()方法，改变模板中超链接的硬编码。
