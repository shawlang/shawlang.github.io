python双下划线的私有变量详解

2015-06-26
Python>Python语法
评论

在学习观察者模式的时候，看到别人的代码中变量命名以双下划线开头。然后在网上搜到这篇文章详细介绍了一下，python对以双下划线命名的变量和方法的处理，总结如下。

私有变量和私有方法并不是真正的私有

在创建一个类A的时候，python会自动的把以双划线命名的变量和方法名进行name mangling，如会把__apple自动转化为_A__apple。

class A:
    def __init__(self):
        self.__apple=5
    def find(self):
        print self.__apple
a=A()
print ','.join(dir(a))
>> _A__apple,__doc__,__init__,__module__,find
#可以看到多了一个_A__apple，却没有__apple

print a.__apple
>> AttributeError: A instance has no attribute '__apple'

print a._A__apple
>> 5
#由此可见python把__apple变成了_A__apple，外部仍然可以通过这种方法访问所谓的私有变量。python算是给成员变量改了一个名字，而访问变量的时候是按照变量名访问的。名若不对，访问就会出问题。

单下划线的变量和方法与正常的变量几乎一样

经测试单下划线的变量是public的，外部可以访问，跟正常的变量一样。为什么说与正常的变量几乎一样，可能是因为python中凡是有下划线的变量名或者方法，都是希望不要被直接使用。

name与main的疑惑

每个模块都有一个name的属性，可以用name获取字符串形式的名字，说不定测试的时候要用到呢。如果在模块当中输入name，就会返回main，表明所写的代码运行在当前模块中。
