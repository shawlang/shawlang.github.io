在github pages免费搭建最新版Hexo博客

2015-06-05
博客搭建>HEXO
评论


本博客用Hexo搭建，虽然过程很折腾，但回过头来又很简单。注意的是多参考官方文档，因为网上教程都有些过时。

Hexo搭建成功以后，只需要在git下面使用$ hexo version即可查看详细的配置参数

工具

  • git 可以到git官网下载
  • node node官网
  • github 需要在github注册账号

步骤

  1. 安装NodeJs,Git以及建立github账户
  2. 在github新建一个Repository，并且命名为[custom name].github.io，并生成Github pages.
  3. 配置SSH KEYS
    • 在git里输入：ssh-keygen -t rsa -C "email adress"
    • 注意这里系统会提示生成保存ssh keys的文件名，直接按Enter跳过，因为到时候把ssh keys的密钥复制到github后，git会连接不上github。原因是github默认保存ssh keys文件名是id_rsa.pub。
    • 这里有个问题，既然是密钥，为什么不同的文件名而密钥会不同，我猜测是安全问题。生成一个文件就会有一个新的密钥，文件名不同密钥也就不同。
    • 继而会提示设置一个“提交项目” 的密码，可填可不填。
    • 如何顺利的话，就会在C:/users/admin/.ssh/id_rsa.pub文件，用编辑器打开并复制。
    • 打开github/settings/SSH keys/Add SSH key,把刚才复制的SSH keys复制进去就可以，title随便取，可不填，保存后github会自动将邮箱地址设置为title

  4. 测试SSH keys
    打开git，以下git@github.com是固定的，不是自己的Email

  8. 配置与部署

    • 找到blog/_config.yml文件，对于站点信息可以自定义配置。

    • 需要注意的2项配置

      1. URL配置：尽可能的写http://shawlang.github.io不要把http://省略掉。因为当开启disqus评论的时候，disqus_url需要的是绝对地址，而非相对地址。不带http的网址，会导致页面disqus评论框不能加载进来。
      2. deploy配置：tpye的类型不要写成github或者gitcafe之类的，直接写成git。Repository直接把仓库在github上显示的名字后面加一个.git就行。

    • 部署：一般是先执行hexo generate或者其简写模式hexo g，然后执行hexo deploy或者其简写模式hexo d。
      对于这两个小步骤，Hexo官网给出了2种形式的简写模式，任选其一即可。

      当然也可以简写，如：hexo g --d和hexo d --

    • 在blog/scaffolds的文件夹可以设置生成文章和页面的默认设置格式。如果新建没一篇文章的时候，没有categories这一项。那么就需要在此打开post.md设置。

    • 新建tags和categories的方法，有2种方式：

      1. tags: [tag1,tag2,tag3]
      2. tags:
        -tag1
        -tag2
        -tag3
