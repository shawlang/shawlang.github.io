python设计模式之观察者模式

2015-06-28
Python>设计模式
评论

第一次真正的学习设计模式，以前总是感觉这个设计模式跟算法一样很缥缈（其实写每一段程序都有算法，程序是数据结构和算法的组合），现在真正的实战一回，感觉很爽！

观察者模式是一种主仆模式

我认为观察者模式是一种主仆模式。仆人（VIEW）从别人那里接收到数据，然后通过控制器（CONTROLLER）数据传给主人（MODEL），并且随时在门外候命。主人一旦把数据处理完了，就通知仆人，仆人再通过控制器把新数据接收到，显示给别人看。

观察者模式的具体实现

虽然是Model是主人，然而仆人很多也是罪，因为时刻受到别人观察。一般来说主人都有一些共性，比如雇佣仆人，辞退仆人，命令仆人也就是有啥事能通知到仆人。

总结

以上只是梳理了一些原理，还有一些细节没有优化。例如把变量设置为私有变量，添加装饰器。
