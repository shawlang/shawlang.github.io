Django自定义中间件

2015-08-14
Django>Django中间件
评论

Django中间件像是一个“审查部门”，客户端发送Request，这时候中间件要么在urls.py之前处理要么是之后处理，中间件处理完成了，views.py才有可能接收到这一请求。等views.py处理完毕后，中间件又得过问一下，再把处理完的Response返回给客户端。

中间件是一个类，主要包含6个方法。

  1. def __init__(self)，这个方法一般类都有，有需要初始化一些属性的时候就会用到这个构造方法。

  2. def process_request(self,request)，request即是客户端发送过来的request对象，而且还在urls.py之前接收到的这个对象。等这个方法处理完了，如果返回None，继续处理其他中间件，然后由urls.py转发给views.py处理；如果返回HttpResponse对象，就没有urls.py和views.py的事了，因为直接返回给客户端了。所以，process_request()方法也叫请求预处理方法。

  3. def process_view(self,request,view_func,view_args,view_kwargs)，视图预处理方法。

request：客户端请求对象
view_func：方法对象本身
view_args：传给视图的参数
view_kwargs：传给视图函数的一个字典的关键字，即dict的keyword

当urls.py真正拿到request请求的时候，process_view()方法又要来干涉一下了，如果返回HttpResponse对象，就直接返回给客户端，没有views.py的事情了；如果返回none，继续处理其他中间件，然后才让views.py处理。

  1. def process_template_response(request,response)，渲染后处理方法。在真正的views.py处理完以后，并且是通过render返回的，如render_to_response(template,context)，那么可以通过这个方法来修改template以及context。注意：这里是修改，所以最后依然要然会一个HttpResponse对象。

  2. def process_response(self,request,response)，应答后处理方法。这个与process_template_response方法类似，都是views.py返回reponse对象后，进行处理，处理完了以后也必须返回一个HttpResponse对象。

  3. def process_exception(self,request,exception)，异常后处理方法。当views.py发生错误的时候或者触发了不可捕获的异常的时候，由这个方法来处理。这个方法可以返回None，也可以返回HttpResponse对象。

开始写自己的中间件

以下中间件是用来检测浏览器是否IE9以下版本，如果是的话，建议用户升级浏览器版本。

步骤一：创建中间件文件

首先在所属app（如myapp）或者project目录下建立一个中间件文件，如mymiddleware.py。

步骤二：定义中间件类

打开mymiddleware.py，给中间件去一个名字：CheckUserAgent

class CheckUserAgent(object):
    def __init__(self):
        pass
    def process_request(self,request):
        agent=request.META['HTTP_USER_AGENT']
        reg=re.search(r'MSIE (\d{1,2})',agent)
        if reg:
            version=reg.group(0).replace("MSIE ","")
            if int(version)<9:
                return HttpResponse("您的浏览器版本是低于IE9，为了更好的用户体验，请您升级至更高版本的浏览器")
    def process_view(self,request,view_func,view_argu,kwarg):
        pass:
  • 这里只需要一个process_request方法就可以。所以中间件类不需要包含所有的方法，方法的多少根据需求来定 *
步骤三：配置中间件

打开settings.py文件，找到”MIDDLE_CLASSES”元组。将中间件加入到这个元组里面。
加入的格式是：就是写入中间件类的具体地址，如这个中间件是放在myapp里面的，路径应该写为：myapp.mymiddleware.CheckUserAgent（[app名]>[文件名]>[中间件类名]）。

MIDDLEWARE_CLASSES = (
‘myapp.middleware.CheckUserAgent’,
‘django.contrib.auth.middleware.AuthenticationMiddleware’,
‘django.contrib.auth.middleware.SessionAuthenticationMiddleware’,
‘django.contrib.messages.middleware.MessageMiddleware’,
‘django.middleware.clickjacking.XFrameOptionsMiddleware’,
‘django.middleware.security.SecurityMiddleware’,
)

注意:配置中间件的顺序很重要的。对于预处理方法，Django会按照中间件配置的顺序从前往后执行；对于后处理方法，则反过来。例如，先执行CheckUserAgent的预处理方法，执行完了以后再执行AuthentionMiddleware的预处理方法，依次执行下去；对于后处理方法，则先执行SecurityMiddleware，然后是XFrameOptionsMiddleware的后处理方法，依次逆序执行下去

关于HttpRequest的常用属性和方法

Django官方文档地址为：https://docs.djangoproject.com/en/dev/ref/request-response/#django.http.HttpRequest.META

1、属性

1) request.scheme 是http或者https
2) request.body request请求body
3) request.path 不包含scheme和域名的页面路径
4) request.path_info 包含路径的端口号
5) request.method 这个常用，有GET 方法和 POST方法
6) request.encoding 编码方法，可以通过这个属性进行更改
7) request.GET/request.POST 字典类型的属性
8) request.COOKIES 字典类型的属性
9) request.FILES 字典类型 注意：只有当request请求方法是POST且enctpye="multipart/form-date"的时候有效，否则request.FILES是一个空的字典
10) request.META 包含所有HTTP请求头信息的字典

CONTENT_LENGTH,CONTENT_TYPE,
HTTP_ACCEPT,HTTP_ENCODING,HTTP_ACCEPT_LANGUAGE
HTTP_HOST,HTTP_REFERER
HTTP_USER_AGENT
QUERY_STRING
REMOTE_ADDR,REMOTE_HOST,REMOTE_USER
REQUEST_METHOD
SERVER_NAME,SERVER_PORT

11) request.user 是AUTH_USER_MODEL实例对象。用来判断用户是否已经登录。如果用户没有登录，request.user被设置成django.contrib.auth.models.AnonymousUser的实例对象。

if request.user.is_authenticated():
    #Do something for logged-in users
else:
    #Do something for anonymous users
  • 注意：只有当中间件AuthenticationMiddleleware开启的时候，这个属性才有效 *

12) request.session 字典类型 只有session开启的时候才有效

13) request.urlconf / request.resolver_match / request/current_app（Django 1.8 新特性）

14) request.get_host() / request.get_port() /

15) request.get_full_path() 得到的值非常完整，如"music/bands/beatles/?content=true"

16) request.build_absolute_uri(location)，得到location的绝对路径，如果location没有设置，location会自动被设置为get_full_path()，最后得到一个绝对路径，如："http://example.com/music/bands/the_beatles/?print=true"

17) request.get_signed_cookie(key,default=RAISE_ERROR,salt=””,age=None)，返回一个COOKIE值。如果signature失效，会抛出一个django.core.signing.BadSignature异常，这时候如果设置了默认的参数，默认的参数值就会返回。

request.get_signed_cookie('name',max_age=60)
#signatureExpired
request.get_signed_cookie('name',False,max_age=60)
#False
request.get_signed_cookie('name',True,max_age=60)
#60

18) request.is_secure() 如果是返回True，则request请求的方法是HTTPS

19) request.is_ajax() 如果返回True，则request是通过XMLHttpRequest发起请求的。

20) request.read(),request.readline(),request.readlines(),request.xreadlines(),request.iter()

总结

很多人把中间件放在Django高级内容里面，个人认为这是其中一项基本配置。程序最高级的部分还是逻辑部分。
